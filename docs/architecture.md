# 系统架构

## 总体设计

Safely 采用云边协同架构，边缘端负责实时视频分析与人群检测，云端负责数据汇聚、业务管理、预警通知和趋势预测。

```
                    ┌─────────────────────────────────────┐
                    │            用户终端                   │
                    │   (浏览器 / 移动端 / 短信通知)        │
                    └──────────────┬──────────────────────┘
                                   │ HTTP / WebSocket
                    ┌──────────────┴──────────────────────┐
                    │              云端集群                 │
                    │                                      │
                    │  ┌─────────┐     ┌──────────────┐   │
                    │  │ anbao   │────▶│   MySQL      │   │
                    │  │ 业务后台 │     └──────────────┘   │
                    │  └────┬────┘     ┌──────────────┐   │
                    │       │          │   Redis      │   │
                    │       ├─────────▶│ (实时数据缓存)│   │
                    │       │          └──────────────┘   │
                    │  ┌────┴────┐     ┌──────────────┐   │
                    │  │ videoIO │────▶│  Hadoop HDFS │   │
                    │  │ 视频模块 │     │ (视频存储)    │   │
                    │  └────┬────┘     └──────────────┘   │
                    │       │                              │
                    │  ┌────┴────┐                         │
                    │  │RabbitMQ │ (消息中间件)             │
                    │  └────┬────┘                         │
                    │       │          ┌──────────────┐   │
                    │       │          │ ARIMA 预测   │   │
                    │       │          │ (定时任务)    │   │
                    │       │          └──────────────┘   │
                    └───────┼─────────────────────────────┘
                            │ RabbitMQ / HTTP
                    ┌───────┼─────────────────────────────┐
                    │       │        边缘端                 │
                    │  ┌────┴────┐                         │
                    │  │ traffic │ (视频分析 / 检测)        │
                    │  └────┬────┘                         │
                    │       │                              │
                    │  ┌────┴────┐                         │
                    │  │  摄像头  │                         │
                    │  └─────────┘                         │
                    └─────────────────────────────────────┘
```

## 模块职责

### 云端

| 模块 | 职责 | 端口 |
|------|------|------|
| anbao | 用户管理、设备管理、区域管理、预警通知、数据统计 | 8080 |
| videoIO | 视频流接收、HDFS 存储、视频回放 | 8080（建议独立部署） |
| html | Web 管理界面（静态资源） | 由 anbao 提供 |
| time_serie_ARIMA | 基于历史数据的人流量趋势预测（定时批处理） | 无（脚本） |

### 边缘端

| 模块 | 职责 |
|------|------|
| traffic | 摄像头视频流采集、人群检测（稀疏/密集场景）、统计数据上报、异常预警触发 |
| flask | 边缘端本地管理 Web 界面 |

## 数据流

### 正常监控流程

```
摄像头 ──▶ traffic(边缘检测) ──HTTP──▶ anbao(统计入库)
                                         │
                                         ▼
                                    MySQL(flowdata表)
                                         │
                                         ▼
                                  ARIMA(定时预测) ──▶ MySQL(forecast表)
```

### 异常预警流程

```
traffic(检测到超阈值) ──HTTP──▶ anbao(/upState?flag=2)
                                    │
                         ┌──────────┼──────────┐
                         ▼          ▼          ▼
                   Redis(实时)  短信通知   WebSocket推送
                                            │
                         ┌──────────────────┘
                         ▼
traffic ──RabbitMQ──▶ anbao(实时人数推送) ──WebSocket──▶ 前端仪表盘
```

### 云端下发指令

```
管理员(Web) ──▶ anbao ──RabbitMQ(fanout)──▶ traffic(修改阈值/请求图片)
```

## 安全架构

详见 [security/overview.md](security/overview.md)。

- 认证：Session（用户端）+ API Key（设备端）
- 传输：建议全站 HTTPS
- 密码：BCrypt 哈希存储
- 防护：CSRF Token、CSP、XSS 转义、路径校验
- 消息：HMAC-SHA256 签名
